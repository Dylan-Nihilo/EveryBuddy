## Design Context

### Users
EveryBuddy 面向重度终端用户，主要是把 Ghostty、Warp、iTerm、tmux、Claude Code、Codex CLI 当成日常工作台的开发者与创作者。他们希望终端不是冷的工具面板，而是带有陪伴感、审美感和一点游戏性的 personal workspace。当前 showcase 不再只是评审稿，而是朝正式宣传官网演进，需要同时承担产品表达、品牌感和功能演示。

### Brand Personality
品牌气质是终端原生、机敏、有收藏欲。三词概括：playful, sharp, ceremonial。界面应同时给人两种感觉：一是“这东西真能工作”，二是“这东西值得把玩”。允许一点游戏化戏剧感，但不能滑向二次元花哨或 debug 面板。

### Aesthetic Direction
视觉方向调整为更现代、更产品化的 terminal-native brand site：冷白与石墨为主，辅以少量带电感的绿色或青色信号色。保留收藏感与游戏化仪式，但不要复古暖色、旧纸感、做旧渐变，也不要普通 SaaS dashboard。页面应该像一款真正要发布的开发者产品官网，同时能展示 EveryBuddy 的卡片收藏态与 live runtime。

### Design Principles
1. 宠物本体永远是主角，UI 只是陈列与烘托，不抢戏。
2. 稀有度、物种、状态变化要比说明文字更先被看到。
3. gallery 要同时表达两种形态：收藏态的 card，与运行态的 tmux sidecar。
4. 终端感来自排版、节奏、实时感和交互动态，不来自粗暴堆砌 monospace 和黑底绿字。
5. 成品必须像经过设计，而不是“把 CLI 输出截图摆进网页”。
